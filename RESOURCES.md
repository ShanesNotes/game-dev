# Godot Resources

Starter set of canonical sources. Will be pruned and expanded as the mission
sharpens. Knowledge for lessons is drawn from here — not from parametric guesses.

## Knowledge

- [Official Godot Docs (4.x)](https://docs.godotengine.org/en/stable/)
  The authoritative reference. Use for: anything — API, node behaviour, concepts.
- [Godot "Your first 2D game" tutorial](https://docs.godotengine.org/en/stable/getting_started/first_2d_game/index.html)
  Official step-by-step "Dodge the Creeps" build. Use for: first end-to-end game.
- [Godot Step-by-step intro](https://docs.godotengine.org/en/stable/getting_started/step_by_step/index.html)
  Nodes, scenes, signals, scripting fundamentals. Use for: core mental model.
- [GDScript reference](https://docs.godotengine.org/en/stable/tutorials/scripting/gdscript/gdscript_basics.html)
  Language syntax. Use for: GDScript questions.
- [C# / .NET in Godot](https://docs.godotengine.org/en/stable/tutorials/scripting/c_sharp/index.html)
  Relevant since this is the mono build. Use for: if we go the C# route.

## Wisdom (Communities)

- [r/godot](https://reddit.com/r/godot)
  Large, active subreddit. Use for: project critique, "why won't this work", showcase.
- [Official Godot Forum](https://forum.godotengine.org/)
  Use for: longer-form help questions.
- [Godot Discord](https://discord.gg/godotengine)
  Use for: real-time troubleshooting.
