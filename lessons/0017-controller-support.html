Controller Support

Analog-stick movement and a gamepad attack button — plus a free fix for the diagonal-speed quirk.

TODAY

One function does most of the work

Right now movement is four ifs reading arrow keys. Godot has a built-in that's better in every way: Input.get_vector(). Hand it your four direction actions and it returns a ready-made Vector2 — and it does three things for you at once:

Analog

A gamepad stick reports how far it's pushed, so a half-tilt = half speed. get_vector preserves that.

Deadzone

Sticks drift slightly at rest; get_vector ignores tiny inputs so your character doesn't wander on its own.

Length-clamped

It never returns longer than 1 — which fixes the diagonal-is-faster bug from Lesson 3, finally and for free.

◈ why the stick already moves you

Godot's built-in ui_left/right/up/down actions come pre-bound to the gamepad's left stick and D-pad. So once movement reads them via get_vector, the controller just works — no Input Map changes needed for movement.

Part A — switch movement to get_vector

In the Player script, replace the movement lines in _physics_process. The four ifs become one line:

GDScript · sprite_2d.gd (_physics_process)
func _physics_process(delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
    update_animation(direction)

    if Input.is_action_just_pressed("attack"):
        start_auto_attack()
Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down") — order is left, right, up, down (negative-x, positive-x, negative-y, positive-y). Returns the movement Vector2 directly — keyboard and stick.
Everything else is unchanged. update_animation still works — with a stick, direction values are fractions (e.g. 0.6), and the abs() comparison handles those fine.

Part B — add a gamepad button to "attack"

1

Project → Project Settings → Input Map. Find your attack action.

2

Click attack's ＋. With a controller connected, press the button you want (e.g. A / cross). Godot records it as a Joypad Button. OK.

3

No controller handy? Use the dropdown in that dialog to pick Joypad Buttons → Bottom Action (Sony Cross, Xbox A…) manually. Close settings.

Because the code already uses the "attack" action (not a specific key), adding the gamepad event is all it takes — no script change.

▶

Plug in a controller and press F5. Left stick moves you (gently push for a slow walk), the face button auto-attacks. Unplug and the keyboard works identically. 🎉 Polish arc complete.
