Why the sprite needs an upgrade

A Sprite2D only knows how to draw a picture. It has no concept of "solid" — that's why it floats through everything. Collision is a physics job, and Godot has dedicated nodes for it. We'll swap the player to one built for exactly this.

Three new nodes

CharacterBody2D

A node built for a player-controlled moving thing that respects collisions. It replaces our plain Sprite2D as the player's root. It gives us two superpowers: a velocity property and a move_and_slide() command.

CollisionShape2D

The invisible hitbox. A node has no size to collide with until you give it a shape (a rectangle, a circle…). Both the player and the walls need one.

StaticBody2D

An immovable solid — perfect for walls. It never moves, but other bodies can't pass through it.

Two new code ideas

velocity

A built-in Vector2 property on CharacterBody2D meaning "how fast and which way I'm moving, in pixels per second." You set it; Godot reads it.

move_and_slide()

A command that moves the body using velocity, and — crucially — stops or slides when it hits something solid. It handles collision for you. (It also applies delta internally, so we drop the * delta.)

We'll also switch from _process to _physics_process — same idea (runs every frame) but on Godot's steady physics clock, which is where movement-with-collision belongs. (docs: physics intro)

Part A — rebuild the player as a body

Right now your tree is Main → Sprite2D (with player.gd). We'll restructure it to:

▸ Main └─ Player (CharacterBody2D — player.gd lives here now) ├─ Sprite2D (the robot picture) ├─ CollisionShape2D (the hitbox) └─ Wall (StaticBody2D) └─ CollisionShape2D (the wall's hitbox)
1

Detach the old script. Right-click your Sprite2D → Detach Script. (We'll re-attach player.gd to the new body in a moment.)

2

Add the body. Right-click Main → Add Child Node → CharacterBody2D → Create. Rename it Player.

3

Make the Sprite a child of Player. In the Scene dock, drag Sprite2D onto Player. It should now sit indented under Player.

4

Add the hitbox. Right-click Player → Add Child Node → CollisionShape2D → Create.

5

Give it a shape. Select CollisionShape2D. In the Inspector, Shape → <empty> → New RectangleShape2D. Drag its orange handles in the viewport until the box roughly covers the robot.

6

Re-attach the script to Player. Select Player → Attach Script → in the dialog set Path to your existing res://player.gd (click the folder icon, pick it) → Load. Now the body runs your code.

Part B — update the script

Open player.gd (double-click it in the FileSystem dock). Replace it with this:

GDScript · player.gd
extends CharacterBody2D

var speed = 200

func _physics_process(delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
extends CharacterBody2D — the script now controls the body, not the sprite. That's why velocity and move_and_slide() are available.
Input.get_vector(...) — same one-liner from Lesson 3. Already normalized, already handles gamepads, now feeding into velocity instead of position.
velocity = direction * speed — set the body's speed & direction. No * delta here — move_and_slide handles timing.
move_and_slide() — moves the body and resolves collisions. Hit a wall and it stops or slides — automatically.

Part C — add a wall & see the hitboxes

1

Add the wall. Right-click Main → Add Child Node → StaticBody2D → rename Wall. Give it a CollisionShape2D child with a New RectangleShape2D, then drag it in the viewport to make a long bar somewhere the robot can run into.

2

Make hitboxes visible at runtime. Top menu → Debug → tick Visible Collision Shapes. Now collision outlines show up while the game runs — so you can see the wall even though it has no picture.

3

Run. F5. Drive into the wall — the robot stops and slides along it instead of passing through. 🎉

◇ notice

Push diagonally into the wall and the robot slides along it rather than sticking. That "slide" is the slide in move_and_slide() — free, and it's what makes movement feel good in real games.
