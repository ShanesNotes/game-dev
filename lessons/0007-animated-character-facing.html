An Animated Character That Faces Where It Walks

The leap from "a sprite that slides" to "a character that's alive." Three new code ideas, paced gently.

The plan

We'll swap the static picture for an AnimatedSprite2D (a sprite that plays frame-by-frame animations), load the character spritesheet I made you, define four walk animations, then write code that plays the right one based on which way you're moving.

◈ I made you a character

In FileSystem → assets you'll find character.png — a 128×128 sheet, 4 frames across, 4 rows down. Row order top-to-bottom: down, left, right, up.

Part A — swap in an AnimatedSprite2D

1

In the Scene dock, delete the old Sprite2D under Player (right-click → Delete, or select + Del).

2

Right-click Player → Add Child Node → AnimatedSprite2D → Create. Leave its name as AnimatedSprite2D — our code will rely on that exact name.

3

With it selected, in the Inspector find Sprite Frames → <empty> → New SpriteFrames. A SpriteFrames panel opens at the bottom.

◈ jargon

SpriteFrames is just a container holding named animations, each a list of frames. One AnimatedSprite2D, many animations inside it.

Part B — create the four walk animations

In the SpriteFrames panel you'll see one default animation called default. We'll make four.

1

Rename default to walk_down (double-click the name). Then use the ＋ Add Animation button to add three more: walk_left, walk_right, walk_up. Names must match exactly — the code uses them.

2

Select walk_down. Click the Add frames from Sprite Sheet button (the grid-with-a-photo icon in the panel's toolbar). Pick assets/character.png.

3

In the slice dialog, set Horizontal = 4 and Vertical = 4. The sheet splits into a 4×4 grid. Click the four cells of the top row (left→right), then Add 4 Frames.

4

Repeat for the others, picking the matching row: walk_left = row 2, walk_right = row 3, walk_up = row 4 (bottom).

5

For each animation, set FPS ≈ 8 and make sure the loop toggle is on. Press the ▶ in the panel to preview the walk cycle.

Part C — the code

Open player.gd. Here's the whole thing — three new ideas explained underneath.

GDScript · player.gd
extends CharacterBody2D

var speed = 200
@onready var anim = $AnimatedSprite2D

func _physics_process(delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
    update_animation(direction)

func update_animation(direction):
    if direction == Vector2.ZERO:
        anim.stop()
        return
    if abs(direction.x) > abs(direction.y):
        if direction.x > 0: anim.play("walk_right")
        else: anim.play("walk_left")
    else:
        if direction.y > 0: anim.play("walk_down")
        else: anim.play("walk_up")

The three new ideas

$AnimatedSprite2D — a reference to a child node by name. The $ means "find my child called AnimatedSprite2D." This is how code reaches other nodes. (That's why the name had to match.)
@onready var anim = ... — grab that node once, when the Player is ready, and keep it in a variable called anim. Now we type anim instead of $AnimatedSprite2D everywhere. @onready waits until the child actually exists.
func update_animation(direction): — your first hand-written function with a parameter. We defined it, then called it on the line update_animation(direction). Splitting logic into named functions keeps _physics_process readable.
else / return — else = "otherwise do this"; return = "stop running this function now." If there's no movement we stop the animation and bail early.
abs(direction.x) > abs(direction.y) — which axis is stronger? abs() strips the minus sign, so we compare amount of horizontal vs vertical movement to decide whether to face sideways or up/down. Because get_vector returns fractional values on a gamepad stick, this comparison handles analog diagonals naturally.
▶

Press F5. Walk in each direction — the character animates and faces the way it moves; stop, and it freezes. 🎉 You now have a real top-down character.
