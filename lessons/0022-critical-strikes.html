One more band in the same roll

No new dice. We still roll one d100 in attack() — we just add a second slice. The table is read in order, so the bands stack up from the bottom: Miss first, then (eventually dodge/parry/block, later), then Crit, and whatever's left is an ordinary Hit.

Crit damage

Melee crits deal 200% — exactly double (reference §5). Note: spell crits are only 150% in Vanilla, but we're swinging steel.

Base crit

Start at ~5%. (Later, an Agility stat will raise it; for now it's a flat base.)

Crit suppression

You lose 1% crit per level the target is above you — higher wolves are slippery.

Part A — crit chance from the level gap

Add a base-crit stat and a chance function. From wow-combat-values.md §5: base − 1% per level above you, floored at 0.

GDScript · sprite_2d.gd
# with the other @export vars:
@export var base_crit = 5.0

func crit_chance(t):
    var c = base_crit - (t.level - level)   # -1% per level the target is above you
    return max(c, 0.0)                       # never below 0
base_crit - (t.level - level) — suppression. vs a same-level wolf = 5%; vs L3 (you're L1) = 5 − 2 = 3%.
max(c, 0.0) — floor at zero so a big level gap can't make crit negative.

Part B — add the crit band to the roll

In attack(), slot Crit between Miss and Hit. The crit deals double damage and flags itself so the wolf can show it big and yellow:

GDScript · sprite_2d.gd — attack()
func attack(t):
    var roll = randf() * 100.0
    var miss = miss_chance(t)
    var crit = crit_chance(t)
    if roll < miss:
        spawn_text_over(t, "Miss", Color.GRAY)
    elif roll < miss + crit:
        t.take_damage(attack_damage * 2, true)   # crit: double damage
    else:
        t.take_damage(attack_damage)            # ordinary hit
elif roll < miss + crit — the crit band sits right after miss. The roll is "below miss" → Miss, "below miss+crit" → Crit, otherwise → Hit. One roll, three outcomes.
take_damage(attack_damage * 2, true) — double damage + a crit flag. The true tells the wolf "this one's a crit, show it loud."

Part C — big yellow crit numbers

First, let the damage-number scene take a size. Update show_text in damage_number.gd:

GDScript · damage_number.gd — show_text
func show_text(s, color, size = 18):
    text = s
    modulate = color
    add_theme_font_size_override("font_size", size)
    var tween = create_tween()
    tween.tween_property(self, "position", position + Vector2(0, -28), 0.6)
    tween.parallel().tween_property(self, "modulate:a", 0.0, 0.6)
    tween.tween_callback(queue_free)

Then have the wolf colour and size its number by the crit flag. Update take_damage and spawn_number in wolf.gd:

GDScript · wolf.gd
func take_damage(amount, is_crit = false):
    health -= amount
    if health < 0: health = 0
    health_bar.value = health
    if is_crit:
        spawn_number(amount, Color.YELLOW, 28)   # loud
    else:
        spawn_number(amount, Color.WHITE, 18)
    if health == 0:
        queue_free()

func spawn_number(amount, color, size = 18):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.show_text(str(amount), color, size)
take_damage(amount, is_crit = false) — the default keeps ordinary hits working (they call it with one argument). Crits pass true.
spawn_number(..., size) now uses show_text — yellow + size 28 for crits, white + 18 otherwise.
