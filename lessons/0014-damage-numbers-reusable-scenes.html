The big idea: build one, stamp out thousands

A damage number is a thing you'll spawn hundreds of times per session. You don't build hundreds by hand — you build one scene, save it as its own file, and instance (stamp out a copy) whenever you need one. This is the move behind bullets, coins, enemies, pickups, and particles in every real game codebase.

◇ this pattern reaches everywhere

The wolf pack (Lesson 18) uses the exact same move — "Save Branch as Scene" applies the instancing pattern to a whole enemy. Learn it once on a damage number; you already know how to make a pack.

preload

Loads a scene file into a constant once at startup, ready to copy: preload("res://damage_number.tscn"). Faster than loading it every time you need one.

instantiate()

Stamps out a fresh, independent copy of that scene — a new node with its own position, colour, and lifetime.

Tween

Animates a property over time. We'll tween the number's position up 28 pixels and its alpha to zero simultaneously, then delete it.

tween_callback(queue_free)

Runs a function when the Tween finishes. We use it to delete the label automatically — no manual cleanup needed.

Part A — build the damage-number scene

1

Top menu → Scene → New Scene.

2

In the Scene dock click Other Node → search Label → Create. Rename the root to DamageNumber.

3

Attach a script: Attach Script → keep the default path (res://damage_number.gd) → Create.

4

Save the scene: Ctrl+S → save as res://damage_number.tscn. This file is now your reusable stamp.

Part B — script the number

Replace the default script with this. Two public functions: show_text is the engine; setup is a thin convenience wrapper for plain numbers.

GDScript · damage_number.gd
extends Label

func setup(amount, color):
    show_text(str(amount), color)   # numbers route through show_text

func show_text(s, color, size = 18):
    text = s
    modulate = color
    add_theme_font_size_override("font_size", size)
    var tween = create_tween()
    tween.tween_property(self, "position", position + Vector2(0, -28), 0.6)
    tween.parallel().tween_property(self, "modulate:a", 0.0, 0.6)
    tween.tween_callback(queue_free)
setup(amount, color) — the convenience wrapper. Converts the integer to a string and passes it to show_text. The wolf and player call this for plain hit numbers.
show_text(s, color, size = 18) — the real engine. Takes any string, any colour, optional font size. This signature matters — "Miss", "Dodge", "Parry" text in later lessons reuses this exact function.
add_theme_font_size_override("font_size", size) — overrides the Label's font size at runtime. The default 18 works for normal hits; crits will pass a larger number (Lesson 22).
tween_property(self, "position", ...) — float up 28px over 0.6s. .parallel() starts the fade at the same time so both animations run simultaneously.
tween_callback(queue_free) — self-destruct when done. Without this you'd accumulate thousands of invisible labels. The callback fires after all tweens finish.
◇ coming later: text combat feedback

Because show_text accepts any string, "Miss", "Dodge", and "Parry" floating text in the Combat Depth arc (Lesson 22) will reuse this exact scene — no new code required. Build it right once.

Part C — spawn numbers when the wolf bites you

The player's take_damage already exists from last lesson. Add the preload constant and a helper, then call it.

GDScript · player.gd — additions
# near the top, with the other constants:
const DAMAGE_NUMBER = preload("res://damage_number.tscn")

# add this helper anywhere in the script:
func spawn_number(amount, color):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.setup(amount, color)

# inside take_damage(), after  health_bar.value = health :
    spawn_number(amount, Color.RED)
DAMAGE_NUMBER.instantiate() — stamp out a fresh copy. Every call creates an independent label with its own position and tween lifecycle.
get_parent().add_child(n) — put it in the world under Main, not under the player. If it were a child of the player, it would move with you as you walk — the number would chase you instead of floating where the hit landed.
n.global_position = global_position + Vector2(-8, -30) — place it just above your head, then setup() hands it the number, colour, and kicks off the float-and-fade.

Part D — the wolf gets numbers too

The same pattern goes in wolf.gd — identical structure, white colour. When you fight back (Lesson 15), the numbers are already there.

GDScript · wolf.gd — additions
# near the top:
const DAMAGE_NUMBER = preload("res://damage_number.tscn")

# helper:
func spawn_number(amount, color, size = 18):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.show_text(str(amount), color, size)

# inside take_damage(), after  health_bar.value = health :
    spawn_number(amount, Color.WHITE)
The wolf calls show_text directly so it can pass a size argument — which lets Lesson 22 add crit flashes (bigger yellow numbers) without changing this signature.

I'M YOUR TEACHER — ASK ME ANYTHING

You built the pattern behind half of all game content — and the wolf's numbers are already wired up for when you swing back. Snags: "Invalid call to setup" → make sure damage_number.gd is attached to the scene root (not a child). Numbers not appearing → confirm the preload path is exactly res://damage_number.tscn and the file is saved there. Numbers follow the player → you used add_child on the wrong node; it must be get_parent() (Main), not self.
