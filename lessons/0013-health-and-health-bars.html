What we're adding

Three pieces fit together:

A health number

The player gets max_health and a current health variable that goes down when hit.

A health bar (UI)

A ProgressBar on a CanvasLayer so it stays fixed on screen (it won't scroll with the camera). It mirrors the health number.

A take_damage() method

One function on the player that subtracts health, updates the bar, and checks for death. The wolf calls it each swing.

◈ the new big idea: nodes calling each other's methods

So far signals let nodes talk. The other way is a direct method call: the wolf already holds target (the player), so it can simply say target.take_damage(swing_damage) — "hey player, take 3 damage." Methods are messages between nodes.

Part A — the on-screen health bar

1

Right-click Main → Add Child Node → CanvasLayer → rename HUD. (A CanvasLayer draws in screen space — fixed, ignores the camera.)

2

Right-click HUD → Add Child Node → ProgressBar → rename HealthBar (exact name).

3

Select HealthBar. Drag it to the top-left of the screen and stretch it to a bar shape (≈ 220 × 26). In the Inspector you can untick Show Percentage if you prefer.

4

Leave Min Value 0 / Max Value 100 for now — our code will set the real max from the player's health.

Part B — give the player health

Open player.gd. Add the health variables, wire up the bar in _ready(), and add the take_damage() and die() functions. Here's the full file — movement and animation are unchanged from Lesson 7:

GDScript · player.gd
extends CharacterBody2D

@export var speed = 200
@export var max_health = 30
var health = max_health

@onready var anim = $AnimatedSprite2D
@onready var health_bar = get_node("../HUD/HealthBar")

func _ready():
    add_to_group("player")
    health_bar.max_value = max_health
    health_bar.value = health

func _physics_process(delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
    update_animation(direction)

func take_damage(amount):
    health -= amount
    if health < 0: health = 0
    health_bar.value = health
    print("Player health: ", health)
    if health == 0:
        die()

func die():
    print("- Memory Eternal -")
    get_tree().reload_current_scene()

func update_animation(direction):
    if direction == Vector2.ZERO:
        anim.stop()
        return
    if abs(direction.x) > abs(direction.y):
        if direction.x > 0: anim.play("walk_right")
        else: anim.play("walk_left")
    else:
        if direction.y > 0: anim.play("walk_down")
        else: anim.play("walk_up")
var health = max_health — start full. One variable that goes down as you're hit.
health_bar = get_node("../HUD/HealthBar") — grab the bar so the player can update it. (Player and HUD are both children of Main, so .. then into HUD.)
_ready() — sync the bar to health at startup: set its max and fill it. Otherwise the bar wouldn't match.
func take_damage(amount): — one argument, the amount. Subtract, clamp at 0, update the bar, die if empty. (A second argument for the attacker arrives in Lesson 16 — not yet.)
get_tree().reload_current_scene() — restart the level on death — a dead-simple respawn.

Part C — make the wolf's bite deal damage

In wolf.gd, change only the swing handler so the bite calls the player's new method:

GDScript · wolf.gd — one function
func _on_swing_timer_timeout():
    if target != null and global_position.distance_to(target.global_position) <= melee_range + 16:
        target.take_damage(swing_damage)     # was: print(...)
⚠ tune the numbers

The wolf's Swing Damage is 3 — with 30 health that's a brisk handful of bites. Select the Wolf and tune Swing Damage in the Inspector to taste; that's exactly why we made it @export.

◈ one argument for now

take_damage(swing_damage) passes exactly one value — the amount. In Lesson 16, when the wolf can fight back, take_damage will gain a second argument (from) so the player can retaliate against the attacker. Don't add that yet — keep it simple.
