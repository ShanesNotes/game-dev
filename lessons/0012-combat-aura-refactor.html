Polling vs. events — the lesson under the lesson

The old wolf polled: every frame it measured distances and decided which of four states to be in. That's a lot of code, and it only entered combat if it reached a tiny 40px melee — which circling defeated.

◇ the better pattern

Don't repeatedly ask "is the player close?" Let a zone tell you the instant they arrive or leave. That's event-driven design — the same Area2D + signals you used for the sword, now with body_exited too. Less code, no failure mode.

The whole brain collapses to one idea: "do I have a target or not?"

player enters aura ──▶ target = player ──▶ move to melee & swing player leaves aura ──▶ target = null ──▶ stop, stop swinging

Part A — build the aura

1

Right-click Wolf → Add Child Node → Area2D → rename AggroArea (exact name).

2

Add a child CollisionShape2D to AggroArea → New CircleShape2D → drag its radius handle out to a generous "aggro" size (≈ 150px — a few wolf-widths).

3

Make the aura see the player. Select AggroArea → Inspector → Collision → Mask: untick 1, tick 2. (The player lives on layer 2 — same fix as the sword.)

4

Connect two signals. With AggroArea selected → Node → Signals: double-click body_entered → connect to Wolf; then double-click body_exited → connect to Wolf. You'll get two new functions in wolf.gd.

◈ keep

Your SwingTimer node and its timeout connection from Lesson 11 stay exactly as they are. We're only swapping the brain.

Part B — the whole new brain

Replace everything in wolf.gd with this. Notice how much shorter it is.

GDScript · wolf.gd
extends CharacterBody2D

@export var speed = 120
@export var melee_range = 40.0
@export var swing_damage = 5

@onready var sprite = $Sprite2D
@onready var swing_timer = $SwingTimer

var target = null   # the player, while they're in our aura

func _physics_process(delta):
    if target != null and global_position.distance_to(target.global_position) > melee_range:
        velocity = (target.global_position - global_position).normalized() * speed
    else:
        velocity = Vector2.ZERO
    move_and_slide()
    face_target()

func _on_aggro_area_body_entered(body):
    if body.name == "Player":
        target = body
        swing_timer.start()        # enter combat

func _on_aggro_area_body_exited(body):
    if body.name == "Player":
        target = null
        swing_timer.stop()         # leave combat

func _on_swing_timer_timeout():
    if target != null and global_position.distance_to(target.global_position) <= melee_range + 16:
        print("The wolf bites you for ", swing_damage, "!")

func face_target():
    var face_x = velocity.x
    if target != null:
        face_x = target.global_position.x - global_position.x
    if face_x != 0:
        sprite.flip_h = face_x < 0
var target = null — the entire state, in one variable. null means "nobody" (not in combat). When the aura sees the player, target becomes the player; when they leave, back to null. No enum needed.
_on_aggro_area_body_entered / _exited — the two events. Enter → grab target + start the swing timer. Exit → drop target + stop the timer. Combat on/off, done.
if target != null and ... > melee_range: — only walk if we have a target and aren't already adjacent. Otherwise stand still. (In melee it stops and lets the swing timer do the work.)
Gone: the enum, the four-way match, aggro_radius, leash_range, home, _ready(). ~40 lines became ~25 — and it can't get stuck in CHASE anymore.
▶

Press F5. Step into the aura — combat starts instantly and the bites tick, regardless of how you move. Walk out of the aura — it stops and goes quiet. Try circling: it follows while you're in range (that's combat), and disengages the moment you leave. 🎉 Reliable, and a third the code.
