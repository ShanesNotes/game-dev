The last band — and it's one-sided

Glancing blows are the level gap turned against you: when you attack something higher level, a chunk of your white swings just skid off for partial damage. They can't be dodged, can't crit, can't be reduced by gear — pure tax for fighting above your weight. Against equal or lower level, glancing simply doesn't exist.

◇ the complete table

Miss → Dodge → Parry → Glancing → Crit → Hit. Glancing slots in right before Crit — so a glancing swing never gets the chance to crit. That's the full Vanilla player attack table, in one roll.

Chance

10% + 10% per level the target is above you (reference §6): +1 = 20%, +2 = 30%. Zero against equal/lower level.

Damage

A random fraction of normal, worse the bigger the gap — roughly 90% at +1, ~80% at +2, down to ~55% vs a +3 boss.

Part A — chance and damage

GDScript · player.gd
func glancing_chance(t):
    var gap = t.level - level
    if gap <= 0:
        return 0.0                     # no glancing vs equal/lower level
    return 10.0 + gap * 10.0          # +1 = 20%, +2 = 30%

func glancing_damage(t):
    var skill_diff = (t.level - level) * 5
    var low = clampf(1.3 - 0.05 * skill_diff, 0.01, 0.91)
    var high = clampf(1.2 - 0.03 * skill_diff, 0.2, 0.99)
    return attack_damage * randf_range(low, high)
if gap <= 0: return 0.0 — glancing is one-sided. It exists only when the target out-levels you; this single guard captures that.
clampf(...) + randf_range(low, high) — the WoW damage formula (reference §6): the bigger the skill gap, the lower the band, and each glance rolls somewhere inside it.

Part B — slot it before Crit

GDScript · sprite_2d.gd — attack()
func attack(t):
    var roll = randf() * 100.0
    var miss = miss_chance(t)
    var dodge = dodge_chance(t)
    var parry = parry_chance(t)
    var glancing = glancing_chance(t)
    var crit = crit_chance(t)
    if roll < miss:
        spawn_text_over(t, "Miss", Color.GRAY)
    elif roll < miss + dodge:
        spawn_text_over(t, "Dodge", Color.CYAN)
    elif roll < miss + dodge + parry:
        spawn_text_over(t, "Parry", Color.ORANGE)
    elif roll < miss + dodge + parry + glancing:
        spawn_text_over(t, "Glancing", Color.GRAY)
        t.take_damage(roundi(glancing_damage(t)))   # reduced, can't crit
    elif roll < miss + dodge + parry + glancing + crit:
        t.take_damage(attack_damage * 2, true)
    else:
        t.take_damage(attack_damage)
"Glancing" label + reduced damage — at our small damage values the number barely shifts, so the label is the tell. WoW shows glancing as a normal-looking hit; the label keeps it legible while you learn.
Position is the point. Glancing is before Crit, so a glance is locked out of critting — exactly the table's priority order.

◈ why we're not coding crushing / the mob table now
