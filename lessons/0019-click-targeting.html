The problem we're solving

Last lesson's get_node_or_null("../Wolf") is a hardcoded contract: "there is exactly one node with this exact name." With a pack, that contract is broken — Wolf2 and Wolf3 can never be targeted.

The fix: instead of the player hunting for a target by node name, wolves volunteer themselves when clicked. The player just stores "whoever told me to target them." No hardcoded paths needed.

input_pickable

A property on CollisionObject2D nodes (which includes CharacterBody2D). When ON, mouse clicks that land on the node's collision shape fire the input_event signal.

input_event signal

Fires whenever the player interacts with a pickable node — including hovers and releases. We filter to a left mouse press in the handler.

get_first_node_in_group("player")

Finds any node tagged "player" in the scene tree. The wolf uses this to call set_target(self) on the player — no hardcoded path to the player node needed either.

Part A — make every wolf clickable in the scene file

We set input_pickable on the wolf scene's root. This is the key decision — and it comes with a real war story.

▲ set it in the scene, not on an instance

This project hit this exact bug: input_pickable was turned on for the Wolf instance in Main's Inspector (a per-instance override) instead of in wolf.tscn. Result: only that one wolf answered clicks. Wolf2 and Wolf3 — which inherited the scene's default of false — ignored every click entirely.

Setting it in the scene file means every instance — including wolves spawned by code later — gets it automatically. The scene is the right place for properties that belong to all wolves.

In the FileSystem dock, double-click wolf.tscn to open it. You are now editing the scene; changes here apply to all instances.

1

Select the Wolf root node. In the Inspector find Collision Object 2D → Input → Pickable and turn it ON.

2

Select the AggroArea child (an Area2D). Find its Pickable and turn it OFF. The aggro aura is a large invisible circle — if it's pickable it swallows clicks before they reach the wolf's body.

3

Re-select the Wolf root → Node dock → Signals → find input_event → double-click → connect to the wolf script as _on_input_event.

4

Ctrl+S to save wolf.tscn. All three wolves in Main now have Pickable ON and the signal connected — without touching Main at all.

Part B — wolf click handler and highlight

Add two functions to wolf.gd:

GDScript · wolf.gd — additions
func _on_input_event(_viewport, event, _shape_idx):
    if event is InputEventMouseButton and event.pressed and event.button_index == MOUSE_BUTTON_LEFT:
        get_tree().get_first_node_in_group("player").set_target(self)

func set_targeted(on):
    sprite.modulate = Color(1, 0.85, 0.4) if on else Color.WHITE
event is InputEventMouseButton and event.pressed and ... LEFT — filter carefully. input_event fires for mouse hovers and button releases too. We want exactly a left-button down event.
get_first_node_in_group("player").set_target(self) — the wolf tells the player "target me." The player has been in the "player" group since Lesson 8 — groups paying off again. self is this specific wolf instance.
set_targeted(on) — gold tint when targeted, white otherwise. Pure visual feedback — the player calls this to set and clear the glow as targets change.

Part C — the Player tracks one target

Replace start_auto_attack and add set_target to player.gd. The key insight: swing() already reads target every tick — we just need a way to set it.

GDScript · player.gd — set_target and updated start_auto_attack
var target = null    # already at the top from L16

func set_target(new_target):
    if is_instance_valid(target):
        target.set_targeted(false)   # un-highlight the old one
    target = new_target
    target.set_targeted(true)        # highlight the new one

func start_auto_attack():
    if is_instance_valid(target) and swing_timer.is_stopped():
        swing()                 # immediate first hit
        swing_timer.start()     # then keep swinging on the tick
set_target(new_target) — the whole swap in five lines. Clear the old glow, point target at the new wolf, light it up. This is all "target switching" is.
start_auto_attack() no longer calls get_node_or_null("../Wolf") — that line is deleted. If there's no target (you haven't clicked anything), pressing attack does nothing. Click a wolf first.

Update take_damage so getting bitten also handles targeting correctly:

GDScript · player.gd — take_damage updated
func take_damage(amount, from = null):
    health -= amount
    if health < 0: health = 0
    health_bar.value = health
    spawn_number(amount, Color.RED)
    if is_instance_valid(from):
        if not is_instance_valid(target):
            set_target(from)       # no target yet -> retaliate at attacker
        if swing_timer.is_stopped():
            swing_timer.start()      # start swinging at your target
    if health == 0:
        die()
Key difference from L16: getting bitten only auto-targets the attacker if you have no target. If you've already clicked Wolf2, a bite from Wolf3 won't steal your focus. WoW-accurate — your deliberate choice takes priority.
set_target(from) instead of target = from — routes through the highlight system. The new wolf glows gold; the old one (if any) goes white. Consistency matters.

Part D — swing() stays unchanged

This is the elegant part: swing() hasn't changed at all since Lesson 16. It just reads target every tick. The moment set_target updates that variable, the next swing lands on the new wolf — without any timer reset.

◇ WoW-accurate weapon rhythm

In WoW, switching targets does not reset your swing timer — your weapon keeps its beat and the next swing just goes to the new enemy. Here, exactly the same thing happens for free: swing_timer keeps running, target changes, done.

◈ auto-attack stays single-target

Hitting all wolves in range at once needs a different approach — an Area2D hitbox that overlaps the "enemies" group. That's Whirlwind, coming later in the combat arc. Auto-attack is always single-target; that's what makes Whirlwind feel special.
