Damage Numbers & Reusable Scenes

Little numbers that float up and fade — and the single most reused skill in game dev: making a scene you can stamp out copies of.

Every hit pops a number: red over your head when you're bitten, white over the wolf when you hit it. Each one floats up, fades, and cleans itself up.

The big skill: a reusable scene

A damage number is a tiny thing you'll spawn hundreds of times. You don't build hundreds by hand — you build one scene, save it as its own file, and instance (stamp out) copies from code whenever you need one.

◇ this is how games are made

Bullets, coins, enemies, particles, UI popups — almost everything is a small reusable scene that gets instanced at runtime. Learn it here on a damage number, and the wolf pack next lesson is the same move.

preload

Loads a scene file into a constant once, ready to copy: preload("res://damage_number.tscn").

instantiate()

Makes a fresh copy of that scene — a new, independent node you can place and configure.

Tween

Animates a property over time. We'll tween the number's position up and its transparency to zero, then delete it.

Part A — build the damage-number scene

1

Top menu → Scene → New Scene.

2

In the Scene dock click Other Node → search Label → Create. Rename the root to DamageNumber.

3

Attach a script to it: Attach Script → Create (path res://damage_number.gd).

4

Save the scene: Ctrl+S → save as damage_number.tscn. This file is now a reusable "stamp."

Part B — script the number

GDScript · damage_number.gd
extends Label

func setup(amount, color):
    text = str(amount)
    modulate = color
    add_theme_font_size_override("font_size", 18)

    var tween = create_tween()
    tween.tween_property(self, "position", position + Vector2(0, -28), 0.6)
    tween.parallel().tween_property(self, "modulate:a", 0.0, 0.6)
    tween.tween_callback(queue_free)
func setup(amount, color): — configure this copy right after we spawn it: what number, what colour.
text = str(amount) — str() turns the number into text for the Label to show.
modulate = color — tints the whole Label that colour (red or white).
create_tween() + tween_property(...) — animate. Move position up 28px over 0.6s; .parallel() fades alpha (modulate:a) to 0 at the same time.
tween_callback(queue_free) — self-destruct when done. Without this you'd pile up thousands of invisible labels.

Part C — spawn one on each hit

Both the player and the wolf will spawn a number inside their take_damage. Add this preload + helper to each script, and one call.

In the Player script (sprite_2d.gd)

GDScript · add to Player
# near the top, with the other vars:
const DAMAGE_NUMBER = preload("res://damage_number.tscn")

# a small helper anywhere in the script:
func spawn_number(amount, color):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.setup(amount, color)

# inside take_damage(), right after  health_bar.value = health :
    spawn_number(amount, Color.RED)

In the Wolf script (wolf.gd)

GDScript · add to Wolf
# near the top:
const DAMAGE_NUMBER = preload("res://damage_number.tscn")

# helper:
func spawn_number(amount, color):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.setup(amount, color)

# inside take_damage(), right after  health_bar.value = health :
    spawn_number(amount, Color.WHITE)
DAMAGE_NUMBER.instantiate() — stamp out a fresh copy of the scene. This is the move you'll reuse for everything.
get_parent().add_child(n) — put it in the world (under Main, not under the wolf) so it stays put and floats while the unit moves on.
n.global_position = ... + Vector2(-8, -30) — place it above the head, then setup() gives it its number, colour, and the float-and-fade animation.
▶

Press F5 and fight. White numbers pop off the wolf as you hit it; red numbers pop off you as it bites. Each drifts up and fades. 🎉 Game-feel unlocked.
