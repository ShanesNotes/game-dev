Two ways to know the player is near

Last lesson the wolf got a brain — a state machine that aggros, chases, and leashes by measuring the distance to the player every frame. That works. But for combat there's a lighter, more reliable tool: instead of the wolf constantly asking "how close is the player?", we let a zone tell it the instant the player arrives or leaves.

◇ events beat polling

The state machine polls — it re-checks distance on every single frame. Event-driven design flips that: an Area2D "aura" around the wolf emits a signal the moment a body enters or leaves it. It's the same Area2D + signals you used for the sword in Lesson 8 — now serving as the wolf's combat trigger. Less code, and it can't "miss" the player.

With the aura doing the detecting, the whole brain collapses to one question: "do I have a target or not?"

player enters aura ──▶ target = player ──▶ move to melee & swing player leaves aura ──▶ target = null ──▶ stop, stop swinging

The new tool: the Timer node

◈ a Timer = a metronome that emits a signal

A Timer node counts down and fires a timeout signal when it hits zero — then (if repeating) starts again. It's how games do anything on a clock: swing timers, global cooldowns, spawn waves, regen ticks. This one signal underpins the whole combat arc.

Ours is the swing timer from WoW: every tick, one auto-attack. We'll start() it when combat begins and stop() it when the fight ends.

Part A — add the swing timer node

1

Right-click the Wolf → Add Child Node → Timer → rename it SwingTimer (exact name — code uses it).

2

Select SwingTimer. In the Inspector set Wait Time = 2. Leave Autostart OFF and One Shot OFF (we want it to repeat, and to start it ourselves).

3

Connect its signal: with SwingTimer selected, go to the Node → Signals tab, double-click timeout(), and Connect it to the Wolf node. Godot makes a function _on_swing_timer_timeout() in wolf.gd.

Part B — add the aggro aura

1

Right-click Wolf → Add Child Node → Area2D → rename AggroArea (exact name).

2

Add a child CollisionShape2D to AggroArea → New CircleShape2D → drag its radius handle out to a generous "aggro" size (≈ 150px — a few wolf-widths).

3

Connect two signals. With AggroArea selected → Node → Signals: double-click body_entered → connect to Wolf; then double-click body_exited → connect to Wolf. You'll get two new functions in wolf.gd.

◈ no layer setup needed

Everything in this scene lives on the default collision layer 1, so the aura detects the player out of the box — leave its Mask alone. The body.name == "Player" check in the script ignores anything else that drifts into the zone.

Part C — the wolf's new brain

Replace everything in wolf.gd with this. It swaps out last lesson's state machine — notice how much shorter it is.

GDScript · wolf.gd
extends CharacterBody2D

@export var speed = 120
@export var melee_range = 40.0
@export var swing_damage = 3

@onready var sprite = $Sprite2D
@onready var swing_timer = $SwingTimer

var target = null   # the player, while they're in our aura

func _physics_process(delta):
    if target != null and global_position.distance_to(target.global_position) > melee_range:
        velocity = (target.global_position - global_position).normalized() * speed
    else:
        velocity = Vector2.ZERO
    move_and_slide()
    face_target()

func _on_aggro_area_body_entered(body):
    if body.name == "Player":
        target = body
        swing_timer.start()        # enter combat

func _on_aggro_area_body_exited(body):
    if body.name == "Player":
        target = null
        swing_timer.stop()         # leave combat

func _on_swing_timer_timeout():
    if target != null and global_position.distance_to(target.global_position) <= melee_range + 16:
        print("The wolf bites you for ", swing_damage, "!")

func face_target():
    var face_x = velocity.x
    if target != null:
        face_x = target.global_position.x - global_position.x
    if face_x != 0:
        sprite.flip_h = face_x < 0
var target = null — the entire combat state, in one variable. null means "nobody" (not in combat). When the aura sees the player, target becomes the player; when they leave, back to null. No enum needed.
_on_aggro_area_body_entered / _exited — the two events. Enter → grab target + start the swing timer. Exit → drop target + stop the timer. Combat on/off, done.
if target != null and ... > melee_range: — only walk if we have a target and aren't already adjacent. Otherwise stand still and let the swing timer do the work. That "stop at melee" is what keeps the wolf from sliding through you.
Gone from last lesson's brain: the enum, the match, aggro_radius, leash_range, home, _ready(). The aura replaces all of it — event-driven beats polling.
▶

Press F5. Step into the aura — combat starts instantly and the bites tick, regardless of how you move. Walk out of the aura and it stops and goes quiet. Try circling the wolf: it follows while you're in range (that's combat) and disengages the moment you leave. 🎉 Reliable combat, a third of the code.
