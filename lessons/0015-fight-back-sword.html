Three new ideas

An input action

A named binding in Godot's Input Map. Instead of checking a specific key, code checks the action — so one action can be bound to a key, a mouse button, and a gamepad button at once.

"just pressed" vs "pressed"

Movement uses is_action_pressed (true the whole time held). An attack should fire once per tap — that's is_action_just_pressed.

A cooldown (stepping stone)

A one-shot Timer + a can_attack flag limits swing rate. Lesson 16 replaces this with a repeating swing timer — but it's worth seeing the simpler form first so the upgrade makes sense.

Part A — define the "attack" action

Create a custom action in the Input Map and bind two events to it: a keyboard/mouse event and a gamepad button. Any device bound to the same action name triggers the same code — no code changes needed per device.

1

Project → Project Settings… → Input Map tab. Type attack in the "Add New Action" box and click Add.

2

Find attack in the list. Click its ＋. Press Space (or left mouse button) → OK. One event bound.

3

Click ＋ again on the same action. With a controller plugged in, press the face button (A/Cross) — Godot records it as a Joypad Button. No controller? Pick Joypad Buttons → Bottom Action (Sony Cross, Xbox A) from the dropdown.

4

Close Settings. The action now has two bindings — keyboard/mouse and gamepad — without a line of code.

◈ one action, many devices

This is Godot's input philosophy: your code says is_action_just_pressed("attack") — it doesn't care how the player pressed it. Adding gamepad support is entirely an Input Map decision, not a scripting one.

Part B — add a cooldown Timer to the Player

1

Select Player → + Add Child Node → Timer → rename it AttackCooldown.

2

In the Inspector: Wait Time = 0.6, tick One Shot ON.

3

Select AttackCooldown → Node → Signals → double-click timeout → connect to Player (creates _on_attack_cooldown_timeout).

Part C — the player script

Movement is already using Input.get_vector since Lesson 3. The new lines are the attack block.

GDScript · player.gd (full file)
extends CharacterBody2D

@export var speed = 200
@export var max_health = 30
@export var attack_damage = 4
@export var attack_range = 60.0
var health = max_health
var can_attack = true

@onready var anim = $AnimatedSprite2D
@onready var health_bar = get_node("../HUD/HealthBar")
@onready var attack_cooldown = $AttackCooldown

const DAMAGE_NUMBER = preload("res://damage_number.tscn")

func _ready():
    add_to_group("player")
    health_bar.max_value = max_health
    health_bar.value = health

func _physics_process(_delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
    update_animation(direction)

    if Input.is_action_just_pressed("attack") and can_attack:
        attack()

func attack():
    can_attack = false
    attack_cooldown.start()
    var wolf = get_node_or_null("../Wolf")
    if wolf != null and global_position.distance_to(wolf.global_position) <= attack_range:
        wolf.take_damage(attack_damage)

func _on_attack_cooldown_timeout():
    can_attack = true

func take_damage(amount):
    health -= amount
    if health < 0: health = 0
    health_bar.value = health
    spawn_number(amount, Color.RED)
    if health == 0:
        die()

func spawn_number(amount, color):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.setup(amount, color)

func die():
    get_tree().reload_current_scene()

func update_animation(direction):
    if direction == Vector2.ZERO:
        anim.stop()
        return
    if abs(direction.x) > abs(direction.y):
        if direction.x > 0: anim.play("walk_right")
        else: anim.play("walk_left")
    else:
        if direction.y > 0: anim.play("walk_down")
        else: anim.play("walk_up")
Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down") — movement hasn't changed since Lesson 3: one call, analog stick or keyboard, diagonal-speed bug already fixed.
is_action_just_pressed("attack") and can_attack — one tap, one swing. just_pressed fires exactly once per button event; can_attack blocks re-entry until the 0.6s cooldown expires.
get_node_or_null("../Wolf") — a stepping stone. This finds the single wolf by its node name. It works for one wolf; Lesson 18 replaces it when the pack arrives.
wolf.take_damage(attack_damage) — the mirror of last lesson. The wolf calls your take_damage; you call its. Combat is two things calling each other's method.
add_to_group("player") in _ready — established since Lesson 8. The wolf already uses this to find you via is_in_group. Keeping it here for clarity.
