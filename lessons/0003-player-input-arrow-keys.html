The new idea: if

Last lesson the robot always moved. A controllable character only moves when a key is held. That word "when" is an if statement: run some lines only if a condition is true.

GDScript · shape of an if
if <something is true>:
    # these indented lines run only then

Same indentation rule as a function: the indented lines belong to the if.

How Godot reads the keyboard

Godot has built-in input actions already mapped to the arrow keys: "ui_up", "ui_down", "ui_left", "ui_right". You ask "is this action being pressed right now?" with:

GDScript
Input.is_action_pressed("ui_right")

That gives back true (held) or false (not held) — exactly what an if wants. (Godot Docs: Input examples)

One more tool: Vector2

Position has two parts: x (left↔right) and y (up↔down). A Vector2 bundles those two numbers into one value, written (x, y). We'll build a direction as a Vector2 — holding right makes (1, 0), up-and-right makes (1, -1).

⚠ y is upside-down

In 2D, y grows downward, so "up" is -1, not +1.

The code

Replace your whole script with this. Read the notes under it as you type.

GDScript · sprite_2d.gd
extends Sprite2D

var speed = 200

func _process(delta):
    var direction = Vector2.ZERO

    if Input.is_action_pressed("ui_right"):
        direction.x += 1
    if Input.is_action_pressed("ui_left"):
        direction.x -= 1
    if Input.is_action_pressed("ui_down"):
        direction.y += 1
    if Input.is_action_pressed("ui_up"):
        direction.y -= 1

    position += direction * speed * delta
var direction = Vector2.ZERO — start each frame at "no movement", i.e. (0, 0). We rebuild direction from scratch every frame based on what's held right now.
if ...("ui_right"): direction.x += 1 — if right is held, push x toward the right. Four independent ifs — holding two at once (right + up) adds both, giving a diagonal automatically.
position += direction * speed * delta — one movement line for all directions. Nothing held → direction is (0,0) → robot stays put.
