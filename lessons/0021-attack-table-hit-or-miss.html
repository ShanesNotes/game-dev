The Attack Table Hit or Miss

One roll of a hundred-sided die decides every swing. Today it decides hit vs miss — and the tougher the wolf, the more you whiff.

TODAY

Every swing rolls the dice. Sometimes you Miss — and a higher-level wolf is harder to hit than a same-level one. This is the literal heart of WoW combat, built one piece at a time.

The one-roll attack table

WoW doesn't roll separately "did I miss? did I crit? did they dodge?" — it makes one roll of a d100 (a number from 0–100) and reads it against a table of outcomes laid end to end. Each outcome owns a slice of the 0–100 range; whatever's left over at the end is an ordinary hit.

◇ we build the table one band at a time

The full order is Miss → Dodge → Parry → Glancing → Block → Crit → Hit (reference doc §0). That's a lot at once — so today we lay down just the first band (Miss) and leave the rest as "hit." Later lessons slot Crit, Dodge, and Parry into the same single roll.

The roll

randf() * 100 gives a number from 0 to 100 — our d100.

The miss band

If the roll lands below your miss chance, it's a Miss. Otherwise (for now) it's a Hit.

The level gap (Δ)

Miss chance grows with how far the target's level is above yours — the foundation from L20, finally doing something.

Part A — miss chance from the level gap

From wow-combat-values.md §3: each level the target is above you = 5 skill points of gap (Δ), and miss is 5% + Δ × 0.1. So vs a same-level wolf = 5%, +1 level = 5.5%, +2 = 6%. Add this to the Player:

GDScript · player.gd
func miss_chance(t):
    var delta = (t.level - level) * 5    # target defense skill − your weapon skill
    return 5.0 + delta * 0.1            # exact for our level range (gap ≤ 2)
(t.level - level) * 5 — the skill gap Δ. t is the target wolf, level is yours. A L3 wolf vs L1 you → Δ = 10.
5.0 + delta * 0.1 — the WoW formula. Δ=0 → 5%, Δ=5 → 5.5%, Δ=10 → 6%. (Bigger gaps get nastier, but our fights never go past Δ=10.)

Part B — roll the die on every swing

Until now swing() just dealt damage when in range. We route that through a new attack() that rolls first:

GDScript · sprite_2d.gd — swing()
func swing():
    if not is_instance_valid(target):
        swing_timer.stop()
        target = null
        return
    swing_sword()
    if global_position.distance_to(target.global_position) <= attack_range:
        attack(target)        # in range -> roll the table

func attack(t):
    var roll = randf() * 100.0          # the d100
    if roll < miss_chance(t):
        spawn_text_over(t, "Miss", Color.GRAY)
    else:
        t.take_damage(attack_damage)     # the leftover band: a hit
var roll = randf() * 100.0 — one roll for the whole table (today: just two bands).
if roll < miss_chance(t) — the miss band sits at the bottom of the range. Land in it → Miss.
else: t.take_damage(...) — everything else is a hit (for now). Crit and dodge will carve their own bands out of this "else" in the next lessons.

Part C — show the "Miss"

A miss needs feedback. We'll reuse the damage-number scene, but teach it to show words, not just numbers. Update damage_number.gd so setup is a thin wrapper over a new show_text:

GDScript · damage_number.gd (full file)
extends Label

func setup(amount, color):
    show_text(str(amount), color)   # numbers route through show_text

func show_text(s, color):
    text = s
    modulate = color
    add_theme_font_size_override("font_size", 18)
    var tween = create_tween()
    tween.tween_property(self, "position", position + Vector2(0, -28), 0.6)
    tween.parallel().tween_property(self, "modulate:a", 0.0, 0.6)
    tween.tween_callback(queue_free)

Then add a helper on the Player that pops text over a target (the wolf you just whiffed on):

GDScript · player.gd
func spawn_text_over(t, s, color):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = t.global_position + Vector2(-8, -30)
    n.show_text(s, color)
setup now calls show_text(str(amount), ...) — your existing damage numbers keep working unchanged; we just gave the scene a second door for words.
spawn_text_over(t, ...) — pops the text over the wolf, not over you, so a "Miss" reads as your swing missing that wolf.
◈ 5% is hard to see — force it to test

At 5–6% you'll miss only now and then. To prove it works, temporarily change miss_chance to return 50.0, run, watch half your swings whiff — then put the real formula back. (Pros do this constantly: crank a value to verify the wiring, then restore.)
