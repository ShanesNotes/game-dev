The Forest: Procedural Generation

Until now you placed every tile and wolf by hand. Today a script builds Elwynn for you — a winding road, scattered trees, wolves prowling the deep grass — and the same seed rebuilds the same forest, every time.

TODAY

Press play and a 56×40 forest generates around you: grass, a road that winds north, dirt shoulders, stone clearings, 28 trees, and extra wolves lurking off the path — your hand-placed pack included. Change one number (the seed) and you get a brand-new forest; keep it and the forest is identical forever.

The idea — a script that owns a node

Every script so far reacted: to input, to timers, to signals. This one builds. We attach a script to the Ground TileMapLayer itself; in _ready() it wipes whatever was painted in the editor (clear()) and paints the whole map from rules. The editor stops being where the world is stored and becomes where the world-builder is configured.

RandomNumberGenerator

Your own private dice. Unlike the global randf(), you can seed it — give it a starting number — and it will produce the exact same sequence of "random" values every run.

seed

The starting number for randomness. Same seed → same rolls → same forest. That's what makes a generated world shareable: "try seed 1337" means something.

FastNoiseLite

A smooth-randomness generator. Where dice jump wildly between values, noise flows — nearby points get similar values. Perfect for natural shapes: undergrowth patches, a road that wanders instead of zigzagging.

map_to_local(cell)

Converts a tile coordinate like (12, 30) into a pixel position (the cell's centre) in the TileMapLayer's space. The bridge between "tile world" and "node world."

Part A — paint the ground

Create forest_generator.gd, attach it to Ground, and start with the setup and the road:

GDScript · forest_generator.gd
extends TileMapLayer

@export var map_width: int = 56
@export var map_height: int = 40
@export var map_seed: int = 1337   # same seed = the exact same forest

var _rng := RandomNumberGenerator.new()
var _noise := FastNoiseLite.new()

func _ready() -> void:
    clear()   # drop any baked tiles — this script owns the map
    _rng.seed = map_seed
    _noise.seed = map_seed
    _noise.frequency = 0.08
    _noise.fractal_octaves = 3
    _paint_ground()
Why not just randf()? The global dice reseed differently every launch — your forest would never be the same twice, and a bug you see would vanish on the next run. Seeded randomness is reproducible randomness.
We name the export map_seed, not seed — GDScript already has a built-in seed() function, and shadowing a built-in name earns you a warning and future confusion.

The road is a single question asked many times: at row y, where is the path? Answer it in one function, then every rule in the generator asks the same oracle:

GDScript · forest_generator.gd
# A sine sway + noise jitter makes the road wander north→south.
func path_x_at(y: int) -> int:
    var wind := sin(y * 0.18) * 3.5 + _noise.get_noise_2d(0, y) * 2.5
    return int(map_width * 0.48 + wind)

func _paint_ground() -> void:
    for y in range(map_height):
        var path_x := path_x_at(y)
        for x in range(map_width):
            var n := _noise.get_noise_2d(x, y)
            var dist_to_path := absi(x - path_x)
            var tile := 0   # lush grass by default
            if dist_to_path < 2:
                tile = 2 if (x + y) % 3 == 0 else 1   # worn road: stone flecks in dirt
            elif dist_to_path < 3 and _rng.randf() < 0.6:
                tile = 1   # dirt shoulder
            if dist_to_path > 4 and n > 0.35:
                tile = 3   # undergrowth pockets, deeper forest
            if dist_to_path > 6 and absf(n) < 0.12:
                tile = 2   # small stone clearings off the road
            set_cell(Vector2i(x, y), 0, Vector2i(tile, 0))
SINGLE SOURCE OF TRUTH

Every placement rule below — trees off the road, wolves in the wild, the player's start beside the path — calls the same path_x_at(). An earlier draft of this generator computed the road three separate ways in three places, and they slowly disagreed: trees kept "randomly" blocking the road. When several rules depend on one fact, make the fact one function.

Part B — trees, from code

You've instanced scenes since Lesson 14; you can also build nodes from nothing. A tree is just a Sprite2D the script creates, dresses, and adds as a child:

GDScript · forest_generator.gd
const TREE = preload("res://assets/tree.png")

func _plant_trees() -> int:
    var placed := 0
    var attempts := 0
    while placed < tree_count and attempts < 200:
        attempts += 1
        var cell := _random_cell()
        if not _is_grass(cell):
            continue   # grass only — keep the road and clearings open
        if absi(cell.x - path_x_at(cell.y)) < 3 and _rng.randf() < 0.7:
            continue   # mostly keep the road's edges clear
        var t := Sprite2D.new()
        t.texture = TREE
        t.position = map_to_local(cell) + Vector2(0, 6)   # trunk sits on the ground
        t.scale = Vector2(_rng.randf_range(0.85, 1.25), _rng.randf_range(0.85, 1.25))
        t.rotation_degrees = _rng.randf_range(-12, 12)
        add_child(t)
        placed += 1
    return placed
The attempts cap matters: if the rules are strict and the dice are unlucky, a bare while placed < tree_count could spin forever. Generators always need an exit.
continue is new: it means "this candidate failed an audition — skip the rest of the loop body and try the next one."

Part C — wolves: the spawner pattern

This was teased all the way back in the pack lessons: spawning enemies from code. It's the same preload → instantiate → add_child move as damage numbers — the pattern behind every wave, dungeon, and respawn you've ever fought:

# inside the spawn loop, after the same grass/road checks as trees:
        if Vector2(cell - start).length() < 8.0:
            continue   # no ambushes on the player's doorstep
        var w := WOLF_SCENE.instantiate()

Because input_pickable lives on the wolf scene root (Lesson 19) and the aggro aura ships inside wolf.tscn (Lesson 18), every spawned wolf is clickable, aggroes, leashes, and levels — for free. This is why we kept moving behaviour into the scene.

Part D — story positions & coordinate spaces

Finally the generator places the player at the south end of the road and moves the pickup sword north — the journey the whole story walks. One subtlety: tiles live in cell coordinates, nodes live in world pixels, and the TileMapLayer itself can be offset. Convert properly:

GDScript · forest_generator.gd
func cell_to_world(cell: Vector2i) -> Vector2:
    return to_global(map_to_local(cell))

func _place_story_positions() -> void:
    if player:
        player.global_position = cell_to_world(_player_start_cell())
    if pickup_sword:
        pickup_sword.global_position = cell_to_world(Vector2i(path_x_at(8) + 1, 8))
MIXED SPACES BITE

map_to_local() answers in the layer's space; a sibling node's global_position lives in world space. Skip the to_global() and everything lands offset by exactly the layer's transform — close enough to look "almost right," which is the worst kind of wrong. This generator's first draft had that bug; the fix is the two-step conversion above.
