The big idea: signals

So far your code asks every frame: "is a key down?" That's fine for input, but imagine asking "did I touch the sword?" 60 times a second for every object — exhausting and messy. Games solve this with signals.

◈ signal = an announcement

A node emits a signal when something noteworthy happens ("a body entered me!", "the timer finished!", "the button was pressed!"). You connect that signal to a function. When the event fires, your function runs — automatically. You stop asking; the game tells you.

This is called event-driven code, and once it clicks you'll see it everywhere in game development.

The detector: Area2D

In Lesson 4 you met solid bodies (CharacterBody2D, StaticBody2D). An Area2D is different — it's not solid, you walk right through it. Its only job is to notice overlaps and emit a signal called body_entered. Perfect for a pickup.

Part A — build the sword pickup

I made you assets/sword.png. We'll build this little tree as a child of Main:

▸ Sword (Area2D — detects the player) ├─ Sprite2D (sword.png) └─ CollisionShape2D (the pickup zone)
1

Right-click Main → Add Child Node → Area2D → rename it Sword.

2

Add a child Sprite2D to Sword; in its Inspector set Texture → Quick Load → assets/sword.png.

3

Add a child CollisionShape2D to Sword; give it a New RectangleShape2D and size it over the sword.

4

In the viewport, drag the whole Sword somewhere out in the grass, away from the player's start.

Part B — script the pickup

1

Select Sword → Attach Script → Create (default path is fine, e.g. sword.gd). Replace its contents with:

GDScript · sword.gd
extends Area2D

func _on_body_entered(body):
    if body.name == "Player":
        print("You found the sword!")
        queue_free()
func _on_body_entered(body): — the function the signal will call. Godot hands it body = whatever walked in. (The name is just a convention; the connection in Part C is what matters.)
if body.name == "Player": — make sure it was the player, not the wolf or a wall. == means "is equal to" (two equals; one equals is assignment).
print(...) — writes a message to the Output panel at the bottom of the editor. Your first debugging tool.
queue_free() — deletes this node (the whole Sword) safely. That's the sword "being picked up" — it disappears.

Part C — connect the signal (the crucial bit)

The function exists, but nothing calls it yet. We wire the Area2D's body_entered signal to it.

1

Select the Sword node. On the right, next to the Inspector tab, click the Node tab, then Signals.

2

Find body_entered(body: Node2D) in the list. Double-click it.

3

In the dialog, the Sword node is selected as the receiver and the method auto-fills as _on_body_entered — exactly our function. Click Connect.

4

A little green "connected" icon appears next to your function in the script. The wire is live.

▶

Press F5, walk onto the sword. It vanishes and "You found the sword!" prints in the Output panel. 🎉 Quest 1 done.

Most common snag: the sword vanishes but nothing prints (or vice-versa) — usually the signal isn't connected (redo Part C), or the player node isn't named exactly Player. If walking over it does nothing at all, check the CollisionShape2D actually has a shape. Want the sword to make the character hold a sword, or play a sound? Great follow-ups — ask.
