Aggro & Leash: The Enemy State Machine

The idea: a state machine

Your wolf currently does one thing forever: chase. Real enemies switch between behaviours depending on the situation. That pattern — "be in exactly one state at a time, and switch on events" — is a state machine, and it's everywhere in games (player states, menus, doors, AI). Learn it once, reuse it forever.

Our wolf gets three states, lifted straight from MMO combat:

IDLE

Sit at home. Do nothing — until the player crosses into the aggro radius. (WoW: standing at spawn.)

CHASE

Pursue the player. But watch the distance from home — stray too far and leash. (WoW: pulled / in combat.)

RETURN

Walk back to the spawn point, ignoring the player. On arrival, drop to IDLE. (WoW: the "evade / reset" you've seen when you kite a mob too far.)

Three new code tools

@export

Puts a variable in the Inspector so you can tune it live without touching code — perfect for aggro_radius and leash_range.

enum

A named list of options. enum State { IDLE, CHASE, RETURN } gives our three states readable names instead of magic numbers.

match

Like a tidy stack of if/else: "match the current state, and run the matching block." Plus distance_to() for measuring gaps, and _ready() to record home once at start.

The script

Replace your wolf.gd with this. It's longer, but it's just the three states spelled out.

GDScript · wolf.gd
extends CharacterBody2D

@export var speed = 120
@export var aggro_radius = 180.0
@export var leash_range = 400.0

@onready var player = get_node("../Player")
@onready var sprite = $Sprite2D

enum State { IDLE, CHASE, RETURN }
var state = State.IDLE
var home = Vector2.ZERO

func _ready():
    home = global_position

func _physics_process(delta):
    match state:
        State.IDLE:
            velocity = Vector2.ZERO
            if global_position.distance_to(player.global_position) <= aggro_radius:
                state = State.CHASE

        State.CHASE:
            var dir = (player.global_position - global_position).normalized()
            velocity = dir * speed
            if global_position.distance_to(home) >= leash_range:
                state = State.RETURN

        State.RETURN:
            var dir = (home - global_position).normalized()
            velocity = dir * speed
            if global_position.distance_to(home) <= 4.0:
                state = State.IDLE

    move_and_slide()
    if velocity.x != 0:
        sprite.flip_h = velocity.x < 0
@export var aggro_radius = 180.0 — shows up in the Inspector. Select the Wolf and you'll see sliders for speed / aggro / leash — tune them while playing, no code edits.
func _ready(): home = global_position — runs once when the wolf enters the scene. We snapshot its starting spot as "home" to leash back to.
match state: — do the block for whichever state we're in. Only one runs each frame. That's the state machine.
distance_to(...) — straight-line distance between two points. IDLE uses it to detect the player (aggro); CHASE uses it to measure the tether to home (leash).
state = State.CHASE etc. — the transitions. Changing state is the whole mechanism — next frame, a different block runs.
◇ the WoW behaviour

Walk near the wolf → it aggros. Lead it on a long chase → it hits leash_range from its spawn and resets, trotting home while ignoring you (try to "kite" it — it won't bite). Back at home it's IDLE and can be pulled again. Classic mob leashing.

▶

Press F5. Approach to pull it, then sprint away past its leash and watch it give up and reset. Select the Wolf in the editor and tweak Aggro Radius / Leash Range in the Inspector to feel the design space. 🎉
