The idea: a sword is just another child node

Your character art has no sword in it — so instead of redrawing every frame, we hang a separate Sprite2D off the Player. Because it's a child, it follows you everywhere automatically — same principle as the Camera in Lesson 6. And because it's its own node, we can rotate it independently for a slash without touching the walk animation at all.

◈ single-target only for now

This lesson comes before the wolf pack (Lesson 18). swing() still hits the one wolf on the target variable. The sword animation and the targeting system are independent — wiring them together doesn't require any changes here.

Child Sprite2D

A second sprite parented to the Player. It rides along with you and can be shown, hidden, and rotated independently of the walk animation.

Offset (pivot)

A Sprite2D rotates around its node's origin. Shifting the texture's Offset lets that origin land on the handle so the blade slashes instead of spinning in place.

SWORD_REST_DEGREES

A named constant for the "at the ready" angle. Having one name instead of 25.0 scattered through the code makes it easy to tune — and ensures equip_sword and the tween return to exactly the same pose.

Part A — add a Sword sprite to the Player

1

Select the Player node → + Add Child Node → Sprite2D. Rename it Sword.

2

In the Inspector, drag assets/sword.png from the FileSystem dock onto the Texture slot.

3

Offset → Y ≈ −16 (about minus-half the image height). This slides the picture up so the node's origin sits at the handle — rotation now happens around the grip, not the blade's centre.

4

Transform → Position ≈ (10, 2) (right hand), Scale ≈ (0.7, 0.7), Rotation ≈ 25°. Nudge until it looks right — the exact values are just aesthetics.

5

Under Visibility, uncheck Visible — the sword starts hidden until you pick up the ground item.

◈ Offset vs Position

Position moves the whole node (where on your body the sword sits). Offset moves just the texture relative to the node's origin — so the origin can land exactly on the handle while the node sits at your hand. Rotation always spins around the origin, which is why this is the correct control for a slash pivot.

Part B — equip it from the pickup

Add the onready reference, the constant, and a small equip_sword() method to the Player. Then call it from sword.gd.

GDScript · player.gd — additions
# near the top, with other @onready vars:
@onready var sword = $Sword

const SWORD_REST_DEGREES = 25.0

# in _ready(), after health bar setup:
    sword.rotation_degrees = SWORD_REST_DEGREES

# anywhere in the script:
func equip_sword():
    sword.visible = true
    sword.rotation_degrees = SWORD_REST_DEGREES
GDScript · sword.gd (the ground pickup)
func _on_body_entered(body):
    if body.name == "Player":
        print("- The blade is thin -")
        body.equip_sword()      # reveal the sword in hand
        queue_free()
body.equip_sword() — the pickup tells the Player to draw its blade. Same node-calls-a-method pattern as take_damage. The ground sword finally does something you can see.
SWORD_REST_DEGREES — one name, used in three places: _ready init, equip_sword, and the tween return. Change the number once and all three update — no magic literals scattered through the code.
◈ deliberate stepping stone

equip_sword() just makes the sprite visible. A future "WeaponData" equip system (where different weapons have different ranges, speeds, and sprites) would replace this — but that system needs Lessons 20+ to motivate it. YAGNI: don't build the system until you need it.

Part C — swing it on every auto-attack

Two tween steps in sequence make the slash: rotate to −40° (fast), then back to rest (slightly slower). Then call it from swing().

GDScript · player.gd — swing_sword() and updated swing()
func swing_sword():
    var t = create_tween()
    t.tween_property(sword, "rotation_degrees", -40, 0.08)          # slash across
    t.tween_property(sword, "rotation_degrees", SWORD_REST_DEGREES, 0.12)  # back to rest

func swing():
    if not is_instance_valid(target):
        swing_timer.stop()
        target = null
        sword.rotation_degrees = SWORD_REST_DEGREES
        return
    swing_sword()                       # animate the slash
    if global_position.distance_to(target.global_position) <= attack_range:
        target.take_damage(attack_damage)
tween_property(sword, "rotation_degrees", -40, 0.08) — first step: the slash. 80ms across. Then the second step plays after it finishes (tweens in a sequence run one after another by default).
tween_property(sword, "rotation_degrees", SWORD_REST_DEGREES, 0.12) — second step: back to rest. 120ms return — slightly slower so the slash reads as snappy but not robotic.
sword.rotation_degrees = SWORD_REST_DEGREES in the dead-target branch — prevents a "stuck-out" pose if the wolf dies mid-slash. Reset to rest immediately.
swing_sword() fires before the distance check — the animation runs on every swing tick regardless of range, so swinging in the air still looks deliberate.
