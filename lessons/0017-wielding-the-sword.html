The idea: a sword is just another node, parented to you

Your character art (the walk frames) has no sword in it — so instead of redrawing it, we hang a separate sword sprite off the Player. Because it's a child, it follows you everywhere automatically (same trick as the Camera in L6). And because it's its own node, we can animate it on its own — a quick rotation arc — without touching the walk animation at all.

Child Sprite2D

A second sprite parented to the Player. It rides along with you and can be shown, hidden, and rotated independently.

Offset (pivot)

A Sprite2D rotates around its origin. Shifting the texture's Offset lets it spin around the handle instead of its middle — so it slashes, not spins.

Tween

From L16 — animate a property over time. Here we tween rotation_degrees out and back for the swing.

Part A — put a sword in the Player's hand

1

Select the Player node → + Add Child Node → Sprite2D. Rename it Sword.

2

In the Inspector, set its Texture to res://assets/sword.png (drag it from the FileSystem dock onto the Texture slot).

3

Offset (under Sprite2D): set Y ≈ −16 (about minus-half the image height). This slides the texture up so the handle sits on the node's origin — now it rotates around the grip.

4

Transform (under Node2D): Position ≈ (10, 2) (the right hand), Scale ≈ (0.7, 0.7), Rotation ≈ 35° (a relaxed "at the ready" tilt). Nudge these until it looks right.

5

Under Visibility, uncheck Visible — the sword starts hidden, and we'll reveal it when you pick it up.

◈ why Offset, not Position, for the pivot

Position moves the whole node (where the sword sits on your body). Offset moves just the picture relative to the node's origin — so the origin can land on the handle while the node sits at your hand. Rotation always spins around the origin, so this is what makes it a slash.

Part B — find the sword, wield the sword

Remember the sword pickup from L8? Right now it just vanishes. Let's make it reveal the held sword. First, a tiny method on the Player:

GDScript · sprite_2d.gd
# with the other @onready vars near the top:
@onready var sword = $Sword

# a small method anywhere in the script:
func equip_sword():
    sword.visible = true

Then have the pickup call it:

GDScript · sword.gd
func _on_body_entered(body):
    if body.name == "Player":
        print("- The blade is thin -")
        body.equip_sword()      # reveal the sword in hand
        queue_free()
body.equip_sword() — the pickup tells the Player to draw its blade. Same node-calls-a-method pattern you used for take_damage; the L8 pickup finally does something you can see.

Part C — swing it on every attack

One Tween makes the slash: rotate the sword across, then back to rest. Add the helper, then call it from swing:

GDScript · sprite_2d.gd
func swing_sword():
    var t = create_tween()
    t.tween_property(sword, "rotation_degrees", -40, 0.08)   # slash across
    t.tween_property(sword, "rotation_degrees", 35, 0.12)    # back to rest
GDScript · sprite_2d.gd — swing()
func swing():
    if not is_instance_valid(target):     # target dead/gone -> stop
        swing_timer.stop()
        target = null
        return
    swing_sword()                       # NEW: animate the slash
    if global_position.distance_to(target.global_position) <= attack_range:
        target.take_damage(attack_damage)
create_tween() + two tween_property calls — they run in sequence: first the fast slash to −40°, then the slower return to 35°. Total ~0.2s, so it finishes well before your next swing.
swing_sword() inside swing() — the visual fires on every swing tick, right alongside the damage. Animation and gameplay, linked.
