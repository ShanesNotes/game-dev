The one idea: everything is a Node

A Godot game is a tree of nodes.

A node is a single small building block that does one job: draw a picture, play a sound, detect a collision, hold a position. You build a game by combining many small nodes into a tree — like LEGO bricks snapped together.

A Scene is just a saved branch of that tree. Your player could be a scene. A coin could be a scene. The whole level is a scene that holds the others. (Godot Docs: Nodes and Scenes)

▸ Main (root node — a Node2D) └─ Sprite2D (draws a picture on screen)

You'll build exactly this. Node2D is an empty anchor that holds a position; the Sprite2D hangs off it and shows an image.

Why this matters for the games you want to make

Zelda, Stardew, Pokémon — every character, tree, chicken, and dialog box is a small scene made of nodes, dropped into a bigger scene. Once nodes-and-scenes clicks, the rest of Godot is just which nodes exist and what they do.

Do it: your first scene on screen

In the Godot window on your right.

1

Create the project. In the Godot Project Manager, click ＋ Create. Name it first-steps, path ~/game-dev/projects/, leave the renderer default, click Create & Edit.

2

Add the root node. In the Scene dock (top-left), click Other Node, search Node2D, Create. Rename it to Main (double-click).

3

Add a Sprite2D. Right-click Main → Add Child Node → search Sprite2D → Create. It now hangs under Main.

4

Give it a picture. With Sprite2D selected, in the Inspector (right) find Texture → click <empty> → Quick Load → pick icon.svg (the Godot robot, ships with every project). The robot appears.

5

Save the scene. Ctrl+S → save as main.tscn. (.tscn = "text scene" — your tree, saved as a file.)

6

Run it. Press F5 (or ▶, top-right). Godot asks for a main scene → Select Current. A game window opens with your robot. 🎉 That window is your game running — close it to return to the editor.
