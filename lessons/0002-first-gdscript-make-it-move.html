What is a script?

In Lesson 1 you built a tree of nodes. But nodes just sit there. To make a node do something, you attach a script — a list of instructions the node follows while the game runs. Think of the node as a robot body, and the script as its brain. (Godot Docs: Creating your first script)

Three words you need first

Function

A named chunk of instructions. When something "calls" the function by its name, every line inside runs, top to bottom. You'll write a function whose name Godot already knows to call.

Property

A value that belongs to a node, like its position. Reading it tells you where the node is; changing it moves the node.

Frame

A game redraws the screen many times per second (each redraw = one frame, ~60/sec). Nudge the position a little every frame and the eye sees smooth motion.

The code you'll write

Read the line-by-line breakdown under it before typing.

GDScript · player.gd
extends Sprite2D

var speed = 200

func _process(delta):
    position.x += speed * delta
extends Sprite2D — "this script controls a Sprite2D node." Added automatically. Lets the script use that node's properties (like position).
var speed = 200 — creates a variable named speed holding 200. A variable is a labelled box you store a value in.
func _process(delta): — defines a function named _process. Godot calls a function with this exact name once every frame, so lines inside run ~60×/sec. (docs)
delta — seconds since the last frame (a tiny number Godot hands you). Multiplying by it makes "200 pixels per second", not per frame.
position.x += speed * delta — the movement. position.x is the horizontal spot; += means "add to what's there." Each frame: move right by speed × delta pixels.
⚠ Indentation matters

The last line is indented (one Tab) inside the function. That indent is how Godot knows the line belongs to _process. Get it wrong and it won't run.

Do it

1

Reopen the Lesson 1 project if closed: godot --path ~/game-dev/projects/first-steps (or pick it in the Project Manager).

2

In the Scene dock, click your Sprite2D node to select it.

3

Click Attach Script (scroll icon with a +, top of the Scene dock). Godot pre-fills the name as sprite_2d.gd — change it to player.gd before clicking Create.

4

The Script editor opens with starter lines. Delete everything and type the 5 lines above. Type it — don't paste. Muscle memory matters when you're learning.

5

Press F5. The robot slides right and off screen. 🎉

6

Experiment: stop, change 200 to 50, run. Then try 600. You just felt what a variable does.

◈ Why player.gd, not sprite_2d.gd?

Godot suggests a name based on the node type. That's a fine default — for a one-file project. As your project grows, you'll have many Sprite2Ds: the player, coins, trees, enemies. A script named after the role (player.gd, coin.gd) tells you what it does at a glance. Build that habit now.
