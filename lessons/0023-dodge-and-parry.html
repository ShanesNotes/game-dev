Avoidance: the defender's bands

Miss is you failing. Dodge and Parry are the wolf succeeding — it actively avoids the blow. In the one-roll table they sit right after Miss and before Crit (reference §0):

◇ the table so far

Miss → Dodge → Parry → Crit → Hit. We're inserting Dodge and Parry into the gap we left. (Glancing and Block come next lesson; they slot in too — same roll, more bands.)

Dodge

Base 5%, +0.5% per level the wolf is above you (reference §4). Smooth and gentle.

Parry

Base 5%, but scales hard with level — and only from the front. Since the wolf chases and faces you, it's always your front to its front.

Part A — the two chance functions

Add these to the Player, alongside miss_chance and crit_chance:

GDScript · player.gd
func dodge_chance(t):
    var gap = max(t.level - level, 0)
    return 5.0 + gap * 0.5        # +0.5% per level above you

func parry_chance(t):
    var gap = max(t.level - level, 0)
    return 5.0 + gap * 3.0        # scales hard: +3 = 14% (front only)
max(t.level - level, 0) — the level gap, floored at 0 (a lower-level wolf doesn't avoid more than base).
Why ×3 for parry? WoW's parry jumps fast — ~8% / 11% / 14% at +1/+2/+3 (reference §4). 5 + gap×3 hits those exactly. Dodge's ×0.5 is the gentle WoW curve.

Part B — slot the bands into the roll

Insert Dodge and Parry between Miss and Crit. Each is checked against a running total of the bands before it:

GDScript · player.gd — attack()
func attack(t):
    var roll = randf() * 100.0
    var miss = miss_chance(t)
    var dodge = dodge_chance(t)
    var parry = parry_chance(t)
    var crit = crit_chance(t)
    if roll < miss:
        spawn_text_over(t, "Miss", Color.GRAY)
    elif roll < miss + dodge:
        spawn_text_over(t, "Dodge", Color.CYAN)
    elif roll < miss + dodge + parry:
        spawn_text_over(t, "Parry", Color.ORANGE)
    elif roll < miss + dodge + parry + crit:
        t.take_damage(attack_damage * 2, true)   # crit
    else:
        t.take_damage(attack_damage)            # hit
Each band stacks on the last. miss + dodge, then miss + dodge + parry, then + crit — the running total walks the 0–100 line in priority order.
Order matters. Avoidance (dodge/parry) is checked before crit — exactly WoW's order. A swing the wolf dodges never gets to "could it crit?".
◈ "front only" — why we don't check facing

In WoW a parry only works against attacks from the front. Our wolf always turns to face its target (its face_target from L9), so you're always at its front — parry always applies, correctly. If we later add wolves you can flank, we'd add a facing check (a dot product) to disable parry from behind. Ask me when you want that.
