Why one wolf was a lie

Your combat secretly assumes there's exactly one enemy. Look at the Player's start_auto_attack: it grabs a target with get_node_or_null("../Wolf") — a hardcoded path to the node named Wolf. Drop a second wolf in and Godot names it Wolf2, a third Wolf3 — that path still only ever finds the first. You can't choose who you're fighting.

◇ the WoW model

In WoW your white auto-attack is single-target: it hits whatever you currently have targeted. You pick a target by clicking it; switch targets and your swings follow. (Hitting everything at once is a separate ability — Whirlwind — which we'll build later.)

Save Branch as Scene

Take a node you already built and split it into its own reusable .tscn file — the L16 instancing trick, applied to your wolf.

input_pickable + input_event

Make a physical node clickable, and run code when it's clicked: the input_event signal on the wolf.

Group

A name you stick on a node so others can find it without a hardcoded path: add_to_group("player") → get_first_node_in_group("player").

Part C — make each wolf clickable

We edit the wolf scene itself so every copy becomes clickable at once. In the FileSystem dock, double-click wolf.tscn to open it (changes here apply to all instances).

1

Select the Wolf root. In the Inspector find Collision Object 2D → Input → Pickable and turn it ON. (This lets mouse clicks "hit" the wolf's body.)

2

Select the AggroArea child and turn its Pickable OFF — otherwise the big invisible aura would swallow your clicks before they reach the wolf.

3

Re-select the Wolf root → Node dock → Signals → find input_event → connect it to the wolf script as _on_input_event.

4

Ctrl+S to save wolf.tscn.

Now add the handler (and a small highlight helper) to the Wolf script:

GDScript · wolf.gd
func _on_input_event(viewport, event, shape_idx):
    if event is InputEventMouseButton and event.pressed and event.button_index == MOUSE_BUTTON_LEFT:
        get_tree().get_first_node_in_group("player").set_target(self)

func set_targeted(on):
    sprite.modulate = Color(1, 0.85, 0.4) if on else Color.WHITE
_on_input_event(...) — runs when this wolf is clicked. The event tells us what happened.
if event is InputEventMouseButton and event.pressed and ... LEFT — filter to a left mouse press. input_event fires for hovers and releases too, so we narrow it down.
get_first_node_in_group("player").set_target(self) — tell the player "target me." We find the player by its group (next part), not a hardcoded path. self = this wolf.
set_targeted(on) — tint the wolf gold when targeted, back to white when not. Pure visual feedback so you can see who you've selected.

Part D — the Player listens, and tracks one target

First, put the Player in a player group so the wolves can find it. Then add the set_target method that swaps the highlight.

GDScript · sprite_2d.gd
func _ready():
    add_to_group("player")        # so any wolf can find us
    health_bar.max_value = max_health
    health_bar.value = health

func set_target(new_target):
    if is_instance_valid(target):
        target.set_targeted(false)   # un-highlight the old one
    target = new_target
    target.set_targeted(true)        # highlight the new one
add_to_group("player") — tags the player so get_first_node_in_group("player") works from any wolf. (You still have var target = null at the top — we keep it.)
set_target(new_target) — the swap. Clear the old target's glow, store the new one, light it up. This is all "switching targets" really is.

Part E — auto-attack the current target

Now the swing just hits target — whoever that currently is. Replace start_auto_attack and swing:

GDScript · sprite_2d.gd
func start_auto_attack():
    if is_instance_valid(target) and swing_timer.is_stopped():
        swing()                 # immediate first hit
        swing_timer.start()     # then keep swinging on the tick

func swing():
    if not is_instance_valid(target):     # target dead/gone -> stop
        swing_timer.stop()
        target = null
        return
    if global_position.distance_to(target.global_position) <= attack_range:
        target.take_damage(attack_damage)
What changed: start_auto_attack no longer hunts for "../Wolf" — it just swings at whatever you've clicked. No target? Pressing attack does nothing; click a wolf first.
swing() reads target fresh every tick — so the moment you set_target a new wolf, the next swing lands on it. That's the redirect you asked for.
WoW-accurate touch: we don't reset the swing timer when you switch targets — in WoW your weapon keeps its rhythm and the next swing just goes to the new enemy. Here, same thing, for free.

One small fix so getting bitten still makes you fight back — update take_damage:

GDScript · sprite_2d.gd — take_damage
func take_damage(amount, from = null):
    health -= amount
    if health < 0: health = 0
    health_bar.value = health
    spawn_number(amount, Color.RED)
    if is_instance_valid(from):
        if not is_instance_valid(target):
            set_target(from)        # no target yet -> retaliate at attacker
        if swing_timer.is_stopped():
            swing_timer.start()       # start swinging at your target
    if health == 0:
        die()
What changed: the old version always did target = from. Now getting bitten only auto-targets the attacker if you had no target — if you've deliberately clicked a wolf, a bite from another won't steal your focus (also WoW-accurate).
