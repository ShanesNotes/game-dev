Why this also fixes the "mirror" bug

Last session we proved the wolf orbit-locks because it forever aims at where you are. Real MMO enemies don't do that — they reach melee range and stop, then auto-attack on a fixed swing timer. So we add a fourth state, COMBAT: the moment the wolf is close enough, it quits chasing and starts swinging. No more endless pursuit, no more mirror.

state flow: IDLE ──(you enter aggro radius)──▶ CHASE ──(reach melee range)──▶ COMBAT ▲ │ └──────── RETURN ◀──(dragged past leash)─────────────┘

The new tool: the Timer node

◈ a Timer = a metronome that emits a signal

A Timer node counts down and fires a timeout signal when it hits zero — then (if repeating) starts again. It's how games do anything on a clock: swing timers, global cooldowns, spawn waves, regen ticks. This one signal underpins the entire combat arc.

This is the swing timer from WoW: every tick, one auto-attack. We'll start() it when combat begins and stop() it when the fight ends.

Part A — add the swing timer node

1

Right-click the Wolf → Add Child Node → Timer → rename it SwingTimer (exact name — code uses it).

2

Select SwingTimer. In the Inspector set Wait Time = 2. Leave Autostart OFF and One Shot OFF (we want it to repeat, and to start it ourselves).

3

Connect its signal: with SwingTimer selected, go to the Node → Signals tab, double-click timeout(), and Connect it to the Wolf node. Godot makes a function _on_swing_timer_timeout() in wolf.gd.

Part B — the upgraded wolf brain

Replace wolf.gd with this. It's your aggro/leash machine plus the COMBAT state and the swing handler.

GDScript · wolf.gd
extends CharacterBody2D

@export var speed = 120
@export var aggro_radius = 180.0
@export var leash_range = 400.0
@export var melee_range = 40.0
@export var swing_damage = 5

@onready var player = get_node("../Player")
@onready var sprite = $Sprite2D
@onready var swing_timer = $SwingTimer

enum State { IDLE, CHASE, COMBAT, RETURN }
var state = State.IDLE
var home = Vector2.ZERO

func _ready():
    home = global_position

func _physics_process(delta):
    var to_player = global_position.distance_to(player.global_position)
    var from_home = global_position.distance_to(home)

    match state:
        State.IDLE:
            velocity = Vector2.ZERO
            if to_player <= aggro_radius:
                state = State.CHASE

        State.CHASE:
            velocity = (player.global_position - global_position).normalized() * speed
            if to_player <= melee_range:
                enter_combat()
            elif from_home >= leash_range:
                state = State.RETURN

        State.COMBAT:
            velocity = Vector2.ZERO          # stand and fight — no more chasing
            if to_player > melee_range + 16:    # you stepped out of melee
                state = State.CHASE
                swing_timer.stop()
            elif from_home >= leash_range:
                state = State.RETURN
                swing_timer.stop()

        State.RETURN:
            velocity = (home - global_position).normalized() * speed
            if from_home <= 4.0:
                state = State.IDLE

    move_and_slide()
    face_target()

func enter_combat():
    state = State.COMBAT
    if swing_timer.is_stopped():
        swing_timer.start()        # begin the swing-timer auto-attacks

func _on_swing_timer_timeout():
    if state == State.COMBAT:
        print("The wolf bites you for ", swing_damage, "!")

func face_target():
    var face_x = velocity.x
    if state == State.COMBAT:
        face_x = player.global_position.x - global_position.x
    if face_x != 0:
        sprite.flip_h = face_x < 0
@onready var swing_timer = $SwingTimer — grab the Timer node you added, so code can start()/stop() it.
State.COMBAT: velocity = Vector2.ZERO — the wolf plants its feet. This single line is what kills the orbit-mirror — in combat it doesn't move toward you at all.
enter_combat() — switch to COMBAT and start the swing timer once (the is_stopped() guard stops us restarting it every frame).
_on_swing_timer_timeout() — fires every 2 seconds while the timer runs. For now it just prints the bite — next lesson it'll subtract health.
to_player > melee_range + 16 — a small buffer so the wolf doesn't flicker between CHASE and COMBAT right at the edge. Step well away and it chases again.
