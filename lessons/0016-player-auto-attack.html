The shift: from GCD to swing timer

Last lesson, one tap = one swing plus a 0.6s lockout. WoW's model is different: once you engage a target you auto-swing on a timer until it's dead or gone. Tapping attack just starts that process. This frees your other buttons — ability keys (Lesson 22+) don't fight the attack button for input.

You already built exactly this for the wolf (a repeating SwingTimer that calls take_damage). Now the player gets the same brain. Combat becomes symmetrical.

target

Who you're currently fighting. Set when you press attack (Lesson 18 adds click-to-pick; for now it's the one wolf). Also set automatically when something bites you.

SwingTimer (repeating)

A Timer with One Shot off. Each tick runs swing(). If the target is in range it hits; if not it waits but keeps the timer going.

is_instance_valid()

The target might die mid-fight and be freed. This checks the reference still points to a live node before we use it — avoids crashes on a dead reference.

Part A — upgrade the Timer

1

Select AttackCooldown. In Node → Signals, find the existing timeout → Player._on_attack_cooldown_timeout connection, right-click it → Disconnect.

2

Rename the node to SwingTimer. In the Inspector: untick One Shot, set Wait Time = 1.5 (your attack speed in seconds).

3

Reconnect: with SwingTimer selected → Node → Signals → double-click timeout → connect to Player (creates _on_swing_timer_timeout).

Part B — the new Player script

Remove the old can_attack / attack() / AttackCooldown logic and replace with the swing timer brain. Movement and animation are unchanged.

GDScript · player.gd (full file)
extends CharacterBody2D

@export var speed = 200
@export var max_health = 30
@export var attack_damage = 4
@export var attack_range = 60.0
var health = max_health
var target = null

@onready var anim = $AnimatedSprite2D
@onready var health_bar = get_node("../HUD/HealthBar")
@onready var swing_timer = $SwingTimer

const DAMAGE_NUMBER = preload("res://damage_number.tscn")

func _ready():
    add_to_group("player")
    health_bar.max_value = max_health
    health_bar.value = health

func _physics_process(_delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
    update_animation(direction)

    if Input.is_action_just_pressed("attack"):
        start_auto_attack()

    # Abilities go here later, e.g.:
    # if Input.is_action_just_pressed("ability_1"): cast_fireball()

func start_auto_attack():
    if not is_instance_valid(target):
        target = get_node_or_null("../Wolf")   # one wolf for now; L18 replaces this
    if is_instance_valid(target) and swing_timer.is_stopped():
        swing()                 # immediate first hit
        swing_timer.start()     # then keep swinging on the tick

func _on_swing_timer_timeout():
    swing()

func swing():
    if not is_instance_valid(target):     # target dead/gone -> stop
        swing_timer.stop()
        target = null
        return
    if global_position.distance_to(target.global_position) <= attack_range:
        target.take_damage(attack_damage)

func take_damage(amount, from = null):
    health -= amount
    if health < 0: health = 0
    health_bar.value = health
    spawn_number(amount, Color.RED)
    if is_instance_valid(from):       # hit by an enemy -> engage it
        target = from
        if swing_timer.is_stopped():
            swing_timer.start()
    if health == 0:
        die()

func spawn_number(amount, color):
    var n = DAMAGE_NUMBER.instantiate()
    get_parent().add_child(n)
    n.global_position = global_position + Vector2(-8, -30)
    n.setup(amount, color)

func die():
    get_tree().reload_current_scene()

func update_animation(direction):
    if direction == Vector2.ZERO:
        anim.stop()
        return
    if abs(direction.x) > abs(direction.y):
        if direction.x > 0: anim.play("walk_right")
        else: anim.play("walk_left")
    else:
        if direction.y > 0: anim.play("walk_down")
        else: anim.play("walk_up")
start_auto_attack() — tapping attack engages. If no target is alive, grab the wolf. If not already swinging, hit once immediately and start the repeating timer.
swing() — one auto-swing. Fires on the first press and every timer tick. Out of range? Skips the hit but keeps the timer — walk closer and it resumes next tick.
is_instance_valid(target) — dead-target guard. When the wolf is queue_free'd, the variable still holds a stale reference. This returns false for freed nodes, so we stop cleanly instead of crashing.
func take_damage(amount, from = null): — default parameter. Callers who don't pass an attacker get null (the wolf without L16 still works). When a wolf passes self, we set it as target and start swinging — automatic retaliation.
# Abilities go here later — the hook. The attack key only starts auto-attack once; every subsequent press does nothing if already swinging. Other keys are completely free for spells.

Part C — the wolf names its attacker

One line change: pass self as the attacker so getting bitten auto-engages that wolf.

GDScript · wolf.gd — _on_swing_timer_timeout (one line)
func _on_swing_timer_timeout():
    if target != null and global_position.distance_to(target.global_position) <= melee_range + 16:
        target.take_damage(swing_damage, self)   # pass self as the attacker
