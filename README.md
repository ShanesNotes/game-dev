# Game Dev — Godot Learning Workspace

A `teach`-skill workspace for learning Godot game development — and a live
exhibit of the method behind it.

**Visiting from X?** Start with **[`GUIDE.md`](GUIDE.md)** — the packaged
process — then browse the course at
**[`lessons/index.html`](lessons/index.html)** (open it in a browser).

## How it's used

From this folder:

```
cd ~/game-dev
claude
/teach godot
```

The `teach` skill drives stateful, multi-session learning. It produces
**lessons** (one tightly-scoped HTML file each, opened in your browser)
grounded in your **mission**. The learner's hands do every step in Godot;
the AI only teaches.

## What's here

| Path                | Purpose                                                         |
|---------------------|-----------------------------------------------------------------|
| `GUIDE.md`          | **The packaged method** — how this workspace works, replicably. |
| `MISSION.md`        | *Why* you're learning — grounds every lesson. **Fill first.**   |
| `TELOS.md`          | The why behind the why. Regenerated by the `/goal` command.     |
| `RESOURCES.md`      | Curated, trusted sources (knowledge + communities).             |
| `NOTES.md`          | Teaching preferences + verified environment facts.              |
| `reference/`        | Pinned research (e.g. WoW 1.12 combat values) lessons cite.     |
| `lessons/`          | Self-contained interactive lessons; `index.html` is the map.    |
| `learning-records/` | What's been learned + locked decisions — the teacher's memory.  |
| `projects/`         | The actual Godot projects, built by hand.                       |

## Commands

- `/goal` — audit the work against the telos; `/goal <idea>` to get a verdict
  on proposed work before doing it. Defined in `.claude/commands/goal.md`.

## Environment

Godot **4.6.2 stable (mono)** at `~/.local/bin/godot`. See `NOTES.md`.
